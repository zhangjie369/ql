🎯 Bing Rewards 自动化脚本 - 多账号支持版


✨ 主要功能

1. 多账号支持
• 从环境变量 bing_ck 读取多个账号的cookie
• 支持多行格式：每行一个账号的cookie，用换行符分隔
• 自动遍历所有账号执行任务

2. 搜索任务
• 电脑搜索：执行30次PC端搜索，每3次检查积分变化
• 移动搜索：执行20次移动端搜索，每3次检查积分变化
• 智能停止：当积分不再增加时自动停止搜索

3. 每日活动任务
• 自动获取并执行每日活动任务
• 支持搜索任务和URL访问任务
• 自动报告任务完成状态

4. 更多活动任务
• 执行优先级为0和7的未完成任务
• 过滤已锁定和已完成的任务
• 支持搜索任务和URL访问任务

5. 积分监控
• 实时监控积分变化
• 显示积分增长情况
• 生成详细的积分报告

6. 统一推送
• 汇总所有账号的执行结果
• 发送统一的通知报告
• 支持多种通知方式（需要notify模块）


⚠️ 重要注意事项

1. Cookie配置

export bing_ck="cookie1\ncookie2\n..."

基本是以：MUID=******开头，但有部分不是
结尾是以：sessionNumber%22%3A22%7D
不是的话，可能cookie没抓对

2. Cookie获取方法
• 登录Microsoft账号
• 访问 https://rewards.bing.com/
• 划重点：访问下面的链接，确定右上角有正确显示账号信息（图三）。
• https://cn.bing.com/?scope=web&FORM=TSASDS
• 回到 https://rewards.bing.com/
• 打开开发者工具 (F12)
• 刷新页面，在Network标签页复制Cookie值

3. 环境要求
• Python 3.6+
• 需要安装 requests 模块
• 可选：安装 notify 模块用于推送通知

4. 执行限制
• 搜索任务有智能停止机制，避免无效搜索
• 任务间有随机延迟，避免被检测
• 账号间有20-40秒随机延迟

5. 错误处理
• 自动跳过无法获取积分的账号
• 任务执行失败时继续处理其他任务
• 详细的错误日志记录

6. 安全考虑
• 不要在代码中硬编码cookie
• 使用环境变量存储敏感信息
• 定期更新cookie，避免过期


📋 使用说明

1. 配置环境变量：将多个账号的cookie按行分隔设置到 bing_ck 变量
2. 安装依赖：pip install requests
3. 运行脚本：python script.py
4. 查看结果：脚本会自动执行所有任务并生成报告

💡 提示：建议在首次使用时先测试单个账号，确认配置正确后再添加多个账号。
